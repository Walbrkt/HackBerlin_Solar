# Screenshots checklist

The main `README.md` references these images. Drop the captures here with these
exact filenames and they'll appear inline.

| File | What to capture |
|---|---|
| `hero.png` | Wide shot — Brandenburg model loaded, several panels placed, HUD visible at top, sidebar visible at right. Camera angle showing roof slope clearly. **The single image that sells the project.** |
| `demo-flow.png` | A side-by-side or 4-panel grid: (1) prompt typed, (2) recommendation card filled, (3) auto-propose result on roof, (4) live cost in HUD. Easiest: take the four shots in the app, paste into Figma/Photoshop with arrows. |
| `free-placement.png` | Close-up on the slope with the green/orange translucent preview panel snapping to the surface. Cursor visible, ideally over a flat patch and a sloped patch back-to-back. |
| `prompt-and-stats.png` | Sidebar focus: prompt textarea filled, recommendation card showing all four numbers, the "Inputs used by the model" expanded with provenance badges visible (`extracted` / `regex` / `defaulted`). |
| `propose-design.png` | Just-after-clicking-Propose. Lots of panels in a tidy grid on one roof slope. Bonus: include a second roof slope nearby that did NOT get filled, to show the "stays on its patch" behavior. |
| `panel-models.png` | One flat panel and one slanted panel on the same roof, ideally on different surface tilts. The visual difference between the two GLB models should be obvious. |
| `live-hud.png` | Top of canvas: HUD showing `placed / target` with the green progress bar, plus the "By panel type" sidebar card showing both flat and slanted rows with subtotals. |

## Tips

- Set your browser to a 16:9 viewport (~1600×900) before shooting — keeps aspect ratios consistent.
- Use the macOS `Cmd+Shift+4` then space-bar to capture just the relevant window/region.
- For the HUD shots, place a few panels first so the progress bar is partway full (pure 0/24 or 24/24 looks boring).
- PNG over JPG so the dark UI background stays clean.

If you want a completely different image set, just rename the files in `README.md` to whatever you have — the layout is forgiving.
